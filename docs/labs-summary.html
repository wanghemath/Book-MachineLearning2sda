A.2 Lab design pattern

Most labs follow a common workflow:

  1. State the modeling question and identify the forecasting information set.
  2. Simulate or load a time-series/sequence dataset.
  3. Visualize the series, dependence structure, and diagnostics.
  4. Fit at least one baseline model before fitting a more advanced model.
  5. Evaluate with chronological or rolling-origin validation.
  6. Interpret residuals, uncertainty, and failure modes.
  7. Use AI assistance only after defining the mathematical task and then verify the output manually.

A.3 Recommended local setup

git clone https://github.com/wanghemath/Book-MachineLearning2sda.git
cd Book-MachineLearning2sda
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
jupyter lab

Alternatively, use Conda:

conda env create -f environment.yml
conda activate ts-sequence-learning
jupyter lab

A.4 Responsible AI checklist for labs

Before submitting any lab, verify the following.

  • The train/validation/test split respects time order.
  • Scaling, imputation, feature engineering, and model selection use only information available at the forecast origin.
  • Every advanced model is compared with a baseline such as naive, seasonal naive, linear AR, ARIMA, or a simple feature model.
  • Residuals or forecast errors are examined, not only point metrics.
  • AI-generated code or explanations are checked against the mathematics and the data-generating process.
  • The final notebook is reproducible from a clean runtime.
